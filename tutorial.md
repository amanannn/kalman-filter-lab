## 1. 问题：传感器为什么不能直接相信？

### 一个真实场景

你有一个扫地机器人，它通过屋顶的摄像头（视觉 SLAM）每 0.1 秒汇报一次自己的位置。可是——

- 第 1 秒，摄像头说：你在 (1.00, 0.00)
- 第 2 秒，摄像头说：你在 (1.05, -0.12)  ← y 方向突然偏了 12 厘米？
- 第 3 秒，摄像头说：你在 (0.98, +0.08)  ← x 方向又倒回去了？

机器人明明是在匀速直线行驶，传感器读数却像喝醉了一样乱跳。

**真实世界中的所有传感器都有噪声**，这些噪声来自：

| 噪声来源 | 例子 |
|----------|------|
| 环境干扰 | 光照变化让摄像头匹配特征点出错 |
| 硬件缺陷 | GPS 芯片自身的精度极限 |
| 量化误差 | 传感器把连续信号变成离散数字时的舍入误差 |
| 传输延迟 | 数据从传感器到处理器的过程中产生的抖动 |

### 结论

**完全相信传感器 → 轨迹跳来跳去，不是真实运动。**
**完全忽略传感器 → 你连自己在哪都不知道。**

卡尔曼滤波做的事：**融合"模型的预测"和"传感器的观测"，得到比任何单方都更准的估计。**

## 2. 直觉：用"不确定性"来分配信任

### 一个日常类比

天气预报说明天 25°C。你有两个信息源：

- **(a) 你自己的猜测：** "现在 20°C，按往年趋势明天大概也差不多" → 但你不是气象专家，**不太确定**。
- **(b) 专业气象台的预报：** 卫星+超级计算机算出来的 → 但预报也有误差，**比较确定但不完美**。

你会怎么决定明天穿什么？你会在心里给两边各分配一个"信任权重"：

> 明天温度 ≈ 你的猜测 × (1 - K) + 气象台预报 × K

其中 K 接近 1（你很信气象台），但你也不会完全忽略自己的判断。

### 卡尔曼滤波的独特之处

**K（卡尔曼增益）不是手动调的，而是自动计算的。** 它基于两条规则：

1. **如果模型预测很不确定（方差大）** → K 自动变大 → 多信传感器
2. **如果传感器噪声很大（方差大）** → K 自动变小 → 多信模型

而且这个 K 在每一步都重新计算——滤波器会"学习"并越来越自信。

### 在 Demo 中你能直观看到

打开 `kalman_animated.gif`：
- **前几帧**：蓝色椭圆（不确定范围）很大 → 滤波器"不知道自己估得对不对"
- **中间帧**：蓝色椭圆明显缩小 → 滤波器越来越自信
- **末尾帧**：蓝色椭圆稳定在一个小范围 → 滤波器收敛了

## 3. 数学准备：你需要知道的三个概念

### 3.1 高斯分布（正态分布）—— "钟形曲线"

```
         ^
         |       ╱ ╲
         |      ╱   ╲        ← 峰值在均值 μ
         |     ╱     ╲
         |    ╱       ╲       ← 宽度由标准差 σ 决定
         |   ╱         ╲
         |  ╱           ╲
         | ╱             ╲
         +────────────────────>
               μ
```

- **均值 μ**：分布的中心，最可能的值
- **方差 σ²**：分布的宽度，越大越"胖"（越不确定）

卡尔曼滤波假设所有噪声都是**高斯分布**的。为什么？因为高斯分布有两个好性质：

1. **加法封闭**：两个高斯分布相加，还是高斯分布
2. **线性变换封闭**：高斯分布 × 矩阵，还是高斯分布

这意味着滤波器只需追踪**均值和方差**两个数字，就能完整描述状态的不确定性。不需要追踪整个概率分布的形状。

### 3.2 协方差矩阵 —— "多维度的不确定性"

一维：一个数字（方差 σ²）就够了。

二维：需要知道的不只是 x 和 y 各自有多不确定（方差），还要知道 **x 和 y 之间有没有关联**（协方差）。

比如：如果 x 偏大了，y 是不是也通常偏大？如果是，它们就**相关**。

协方差矩阵 P 是一个正方形的矩阵：

```
        x方差    xy协方差
P  =  [ σ²x     σxy    ]
       [ σxy    σ²y    ]
```

在我们的 4 维状态 `[px, py, vx, vy]` 中，P 是一个 4×4 矩阵，它告诉滤波器：

- 我对 px 的估计有多不确定？
- 我对 vy 的估计有多不确定？
- px 和 vx 之间有相关吗？（通常有——位置偏右时，速度往往也偏右）

### 3.3 矩阵乘法 —— "线性变换"

如果你对矩阵乘法不熟，这里给一个最简解释：

```python
# 2×2 矩阵 × 2×1 向量 = 2×1 向量
[[a, b],    ×  [x,     =  [a*x + b*y,
 [c, d]]       y]         c*x + d*y]
```

矩阵乘法本质是**加权求和**。在卡尔曼滤波中，每一步操作都可以理解为"对当前信息做一次加权求和"。

## 4. 状态空间模型：用数学描述运动

### 4.1 状态向量 x

我们用一个 4 维向量描述物体在某一时刻的完整状态：

```
x = [px, py, vx, vy]ᵀ
     │    │   │   │
     │    │   │   └── y 方向速度 (米/秒)
     │    │   └────── x 方向速度 (米/秒)
     │    └────────── y 方向位置 (米)
     └─────────────── x 方向位置 (米)
```

### 4.2 运动模型（预测）

如果物体以恒定速度运动，从时刻 k 到时刻 k+1：

```
px_{k+1} = px_k + vx_k × dt     ← 旧位置 + 速度 × 时间
py_{k+1} = py_k + vy_k × dt
vx_{k+1} = vx_k                  ← 速度不变（匀速假设）
vy_{k+1} = vy_k
```

写成矩阵形式就是状态转移矩阵 F（代码第 103-108 行）：

```
F = [[1, 0, dt, 0],
     [0, 1, 0, dt],
     [0, 0, 1,  0],
     [0, 0, 0,  1]]
```

一步预测：`x_new = F @ x_old`

### 4.3 观测模型（传感器）

我们的传感器只能测位置，测不了速度。所以观测向量只有 2 维：

```
z = [px_observed, py_observed]ᵀ
```

状态到观测的映射用观测矩阵 H（代码第 110-113 行）：

```
H = [[1, 0, 0, 0],
     [0, 1, 0, 0]]
```

含义：`z = H @ x` → "从 4 维状态中只取前两个（位置），忽略后两个（速度）"

### 4.4 噪声模型

**过程噪声 Q（代码第 116-117 行）：**
- 真实世界中，物体不会完美匀速——可能有风、地面不平、电机误差
- Q 矩阵量化了"运动模型和真实物理之间的偏差"
- Q 越大 → 滤波器越不信任模型的预测

**测量噪声 R（代码第 119-120 行）：**
- 传感器的固有误差
- R 矩阵量化了"传感器的不可靠程度"
- R 越大 → 滤波器越不信任传感器的读数

## 5. 卡尔曼滤波的五个公式（逐条拆解）

每一步滤波 = **预测**（公式 1-2）+ **更新**（公式 3-5）

### 阶段一：预测（我们猜物体现在在哪）

#### 公式 1：状态预测

```
x̂ₖ|ₖ₋₁ = F · x̂ₖ₋₁|ₖ₋₁
```

读法："用上一时刻的最佳估计 x̂ₖ₋₁|ₖ₋₁，通过运动模型 F，预测当前时刻的状态 x̂ₖ|ₖ₋₁"

| 符号 | 含义 |
|------|------|
| x̂ₖ|ₖ₋₁ | 在 k-1 时刻看 k 时刻的预测值（先验估计） |
| x̂ₖ₋₁|ₖ₋₁ | k-1 时刻的最佳估计（后验估计） |
| F | 状态转移矩阵，编码了运动模型 |

**代码：** `kalman_tracker.py` 第 140 行 → `self.x = self.F @ self.x`

#### 公式 2：协方差预测

```
Pₖ|ₖ₋₁ = F · Pₖ₋₁|ₖ₋₁ · Fᵀ + Q
```

预测之后，不确定性**一定会增加**——因为模型不完美（Q），预测得越远越不确定。

**直觉**：你开车时闭眼 0.1 秒 → 你基本知道车在哪。闭眼 5 秒 → 你可能已经偏离车道了。不确定性随时间增长。

**代码：** `kalman_tracker.py` 第 141 行 → `self.P = self.F @ self.P @ self.F.T + self.Q`

---

### 阶段二：更新（传感器读数来了，修正预测）

#### 公式 3：卡尔曼增益

```
K = Pₖ|ₖ₋₁ · Hᵀ · (H · Pₖ|ₖ₋₁ · Hᵀ + R)⁻¹
```

这是卡尔曼滤波的灵魂。K 决定了"信模型"和"信传感器"之间的权重分配。

**极限情况分析：**

- 如果 **R → 0**（传感器完美无误）→ `K ≈ Hᵀ · (H · Hᵀ)⁻¹` → 完全相信传感器
- 如果 **P → 0**（模型极其确定）→ `K ≈ 0` → 忽略传感器，完全相信模型预测
- 正常情况：K 在两者之间自动调节

**代码：** `kalman_tracker.py` 第 156-158 行 → `K = self.P @ self.H.T @ np.linalg.inv(S)`

#### 公式 4：状态更新

```
x̂ₖ|ₖ = x̂ₖ|ₖ₋₁ + K · (zₖ - H · x̂ₖ|ₖ₋₁)
                      └──────┬──────┘
                          创新（残差）
```

**残差 (innovation)** = 传感器实际读数 − 我们预测传感器应该读到的值。

- 如果残差大 → 说明我们猜错了，需要大幅修正
- 如果残差小 → 说明我们猜得准，小幅修正即可

K 决定了"用多大的力度去修正"。

**代码：** `kalman_tracker.py` 第 160 行 → `self.x = self.x + K @ y`

#### 公式 5：协方差更新

```
Pₖ|ₖ = (I - K · H) · Pₖ|ₖ₋₁
```

获得传感器信息后，不确定性**一定会降低**。降多少？看卡尔曼增益 K。

**代码：** `kalman_tracker.py` 第 162 行 → `self.P = (np.eye(4) - K @ self.H) @ self.P`

---

### 五公式总结（一页纸）

```
预测：
  ① x = F·x              "用模型猜下一时刻状态"
  ② P = F·P·Fᵀ + Q      "预测的不确定性增加"

更新：
  ③ K = P·Hᵀ·(H·P·Hᵀ + R)⁻¹   "算权重：信模型还是信传感器"
  ④ x = x + K·(z - H·x)        "用传感器数据修正状态"
  ⑤ P = (I - K·H)·P            "修正后不确定性降低"
```

## 6. 代码走读：公式是如何变成代码的

### 6.1 项目结构

```
kalman_tracker.py
├── generate_ground_truth()     ← 生成真实轨迹（上帝视角的答案）
├── simulate_observations()     ← 给真实位置加噪声，模拟传感器
├── class KalmanFilter          ← 滤波器核心
│   ├── __init__()              ← 初始化 F, H, Q, R, P, x
│   ├── predict()               ← 公式 ① ②
│   ├── update()                ← 公式 ③ ④ ⑤
│   └── step()                  ← 一次完整的 predict + update
├── draw_static_figure()        ← 生成静态 PNG
├── draw_animated_gif()         ← 生成动画 GIF
└── print_metrics()             ← 打印终端摘要
```

### 6.2 初始化做了什么

```python
# 代码第 95-127 行

self.F = [[1,0,dt,0], [0,1,0,dt], [0,0,1,0], [0,0,0,1]]  # 匀速运动模型
self.H = [[1,0,0,0], [0,1,0,0]]                          # 只看位置，不看速度
self.Q = diag([0.01, 0.01, 0.05, 0.05])                  # 过程噪声
self.R = diag([0.45², 0.45²])                            # 测量噪声

self.x = [0, 0, 0, 0]ᵀ   → 初始猜测：在原点，速度为零
self.P = I₄ × 0.5         → 初始不确定：半信半疑
```

初始 P 设得偏大（0.5）的好处：让滤波器在前几步快速改变估计值，而不是固执地停留在错误的初始猜测上。

### 6.3 一步滤波的完整流程

```python
# 代码第 164-167 行

def step(self, z):
    self.predict()    # ① F → x    ② F,P,Q → P
    self.update(z)    # ③ P,H,R → K   ④ K,残差 → x   ⑤ K,H → P
    # 记录结果用于可视化
    self.estimates.append(x[:2])      # 只记录位置
    self.covariances.append(P.copy()) # 记录完整协方差
```

### 6.4 不确定性椭圆的原理

```python
# 代码第 182-195 行

def uncertainty_ellipse(cov_2x2, n_std=1.0):
    eigvals, eigvecs = np.linalg.eigh(cov_2x2)  # 特征分解
    angle = arctan2(eigvecs[1,0], eigvecs[0,0])  # 椭圆方向
    width, height = 2 × n_std × sqrt(eigvals)    # 椭圆尺寸
    return width, height, angle
```

协方差矩阵的**特征值** = 不确定性在各个方向上的"半径"
协方差矩阵的**特征向量** = 不确定性的"伸展方向"

1σ 椭圆 = 物体有 68% 的概率落在这个椭圆内。

### 6.5 误差度量：RMSE

```
RMSE（均方根误差）= sqrt( 平均( (估计值 - 真实值)² ) )
```

代码第 268-271 行计算并打印 RMSE。在 Demo 中：

- **观测 RMSE ≈ 0.57m** — 你如果直接相信传感器，平均误差 57 厘米
- **滤波 RMSE ≈ 0.25m** — 用了卡尔曼滤波，平均误差降到 25 厘米
- **误差降低 ≈ 56%** — 滤波效果显著

## 8. 常见问题

### Q1: 卡尔曼滤波和"简单取平均"有什么区别？

简单取平均：对过去所有传感器读数取平均 → 平滑但滞后严重。

卡尔曼滤波：不仅用历史信息，还用**运动模型**预测未来 → 既平滑又无滞后。

### Q2: P 矩阵在什么时候会发散（越来越大）？

- 如果 **Q 太大而 R 太小** → 滤波器过分相信传感器，P 不稳定
- 如果 **真实运动违背了模型假设** → 比如物体突然以远大于 Q 预估的速度拐弯
- **数值不稳定** → 长期运行中协方差可能因为浮点精度问题变成非正定，需要用 Joseph 形式或平方根滤波解决

### Q3: 为什么叫"滤波"而不叫"预测"？

历史上 "filter" 这个词来自信号处理。卡尔曼滤波确实可以同时做**滤波**（用当前及过去的观测估计当前状态）、**预测**（用过去的观测预测未来状态）和**平滑**（用全部观测估计过去某时刻的状态）。我们的 Demo 做的是"滤波"。

### Q4: 卡尔曼滤波和深度学习有什么区别？

| | 卡尔曼滤波 | 深度学习 |
|---|---|---|
| 需要大量数据 | 不需要 | 需要 |
| 需要运动模型 | 需要（手动设计） | 不需要（从数据中学习） |
| 可解释性 | 完全可解释 | 黑箱 |
| 计算量 | 极低 | 高 |
| 适合场景 | 实时状态估计、融合 | 模式识别、端到端控制 |

实际机器人系统中往往**两者结合使用**——比如用深度学习做目标检测，用卡尔曼滤波做检测结果的追踪和预测。

## 9. 下一步学什么

当你能够回答以下问题，就说明掌握了基础：

1. **卡尔曼滤波的输入和输出分别是什么？**
2. **F、H、Q、R 四个矩阵各自代表什么？**
3. **卡尔曼增益 K 由什么决定？它如何影响滤波行为？**
4. **P 矩阵的变化规律是什么？（预测增大、更新减小）**
5. **什么时候卡尔曼滤波会失效？**

### 进阶路线

```
线性卡尔曼滤波（本文）
       │
       ├─→ 扩展卡尔曼滤波 (EKF)
       │   处理非线性运动（旋转、弹道）
       │   核心：用泰勒展开对非线性函数做局部线性化
       │
       ├─→ 无迹卡尔曼滤波 (UKF)
       │   比 EKF 精度更高，不需要求导
       │   核心：用 sigma 点采样来传递分布
       │
       └─→ 粒子滤波 (Particle Filter)
           处理任意分布（非高斯）
           核心：用大量加权粒子来表示概率分布
           典型应用：Monte Carlo Localization (MCL)
                   └── 机器人"我在哪？"问题的标准解法
```
