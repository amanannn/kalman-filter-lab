## 1. 问题：传感器为什么不能直接相信？

### 一个真实场景

你有一个扫地机器人，它通过屋顶的摄像头（视觉 SLAM）每 0.1 秒汇报一次自己的位置。可是——

- 第 1 秒，摄像头说：你在 (1.00, 0.00)
- 第 2 秒，摄像头说：你在 (1.05, -0.12) <- y 方向突然偏了 12 厘米？
- 第 3 秒，摄像头说：你在 (0.98, +0.08) <- x 方向又倒回去了？

机器人明明是在匀速直线行驶，传感器读数却像喝醉了一样乱跳。

**真实世界中的所有传感器都有噪声**，这些噪声来自：

| 噪声来源 | 例子 |
|----------|------|
| 环境干扰 | 光照变化让摄像头匹配特征点出错 |
| 硬件缺陷 | GPS 芯片自身的精度极限 |
| 量化误差 | 传感器把连续信号变成离散数字时的舍入误差 |
| 传输延迟 | 数据从传感器到处理器的过程中产生的抖动 |

### 结论

**完全相信传感器 -> 轨迹跳来跳去，不是真实运动。**
**完全忽略传感器 -> 你连自己在哪都不知道。**

卡尔曼滤波做的事：**融合"模型的预测"和"传感器的观测"，得到比任何单方都更准的估计。**

## 2. 直觉：用"不确定性"来分配信任

### 一个日常类比

天气预报说明天 25 deg C。你有两个信息源：

- **(a) 你自己的猜测：** "现在 20 deg C，按往年趋势明天大概也差不多" -> 但你不是气象专家，**不太确定**。
- **(b) 专业气象台的预报：** 卫星+超级计算机算出来的 -> 但预报也有误差，**比较确定但不完美**。

你会怎么决定明天穿什么？你会在心里给两边各分配一个"信任权重"：

> 明天温度 ~= 你的猜测 x (1 - K) + 气象台预报 x K

其中 K 接近 1（你很信气象台），但你也不会完全忽略自己的判断。

### 卡尔曼滤波的独特之处

**K（卡尔曼增益）不是手动调的，而是自动计算的。** 它基于两条规则：

1. **如果模型预测很不确定（方差大）** -> K 自动变大 -> 多信传感器
2. **如果传感器噪声很大（方差大）** -> K 自动变小 -> 多信模型

而且这个 K 在每一步都重新计算——滤波器会"学习"并越来越自信。

### 在 Demo 中你能直观看到

打开 `kalman_animated.gif`：
- **前几帧**：蓝色椭圆（不确定范围）很大 -> 滤波器"不知道自己估得对不对"
- **中间帧**：蓝色椭圆明显缩小 -> 滤波器越来越自信
- **末尾帧**：蓝色椭圆稳定在一个小范围 -> 滤波器收敛了

## 3. 数学准备：你需要知道的三个概念

### 3.1 高斯分布（正态分布）—— "钟形曲线"

```
         ^
         |       /\
         |      /  \        <-- 峰值在均值 mu
         |     /    \
         |    /      \       <-- 宽度由标准差 sigma 决定
         |   /        \
         |  /          \
         | /            \
         +-------------------->
 mu
```

- **均值 mu**：分布的中心，最可能的值
- **方差 sigma^2**：分布的宽度，越大越"胖"（越不确定）

卡尔曼滤波假设所有噪声都是**高斯分布**的。为什么？因为高斯分布有两个好性质：

1. **加法封闭**：两个高斯分布相加，还是高斯分布
2. **线性变换封闭**：高斯分布 x 矩阵，还是高斯分布

这意味着滤波器只需追踪**均值和方差**两个数字，就能完整描述状态的不确定性。不需要追踪整个概率分布的形状。

### 3.2 协方差矩阵 —— "多维度的不确定性"

一维：一个数字（方差 sigma^2）就够了。

二维：需要知道的不只是 x 和 y 各自有多不确定（方差），还要知道 **x 和 y 之间有没有关联**（协方差）。

比如：如果 x 偏大了，y 是不是也通常偏大？如果是，它们就**相关**。

协方差矩阵 P 是一个正方形的矩阵：

```
 x方差 xy协方差
P = [ sigma^2x sigmaxy ]
 [ sigmaxy sigma^2y ]
```

在我们的 4 维状态 `[px, py, vx, vy]` 中，P 是一个 4 x 4 矩阵，它告诉滤波器：

- 我对 px 的估计有多不确定？
- 我对 vy 的估计有多不确定？
- px 和 vx 之间有相关吗？（通常有——位置偏右时，速度往往也偏右）

### 3.3 矩阵乘法 —— "线性变换"

如果你对矩阵乘法不熟，这里给一个最简解释：

```python
# 2 x 2 矩阵 x 2 x 1 向量 = 2 x 1 向量
[[a, b], x [x, = [a*x + b*y,
 [c, d]] y] c*x + d*y]
```

矩阵乘法本质是**加权求和**。在卡尔曼滤波中，每一步操作都可以理解为"对当前信息做一次加权求和"。

## 4. 状态空间模型：用数学描述运动

### 4.1 状态向量 x

我们用一个 4 维向量描述物体在某一时刻的完整状态：

```
x = [px, py, vx, vy]^T
 | | | |
 | | | `--- y 方向速度 (米/秒)
 | | `------- x 方向速度 (米/秒)
 | `----------- y 方向位置 (米)
 `---------------- x 方向位置 (米)
```

### 4.2 运动模型（预测）

如果物体以恒定速度运动，从时刻 k 到时刻 k+1：

```
px_{k+1} = px_k + vx_k x dt <- 旧位置 + 速度 x 时间
py_{k+1} = py_k + vy_k x dt
vx_{k+1} = vx_k <- 速度不变（匀速假设）
vy_{k+1} = vy_k
```

写成矩阵形式就是状态转移矩阵 F（代码第 103-108 行）：

```
F = [[1, 0, dt, 0],
 [0, 1, 0, dt],
 [0, 0, 1, 0],
 [0, 0, 0, 1]]
```

一步预测：`x_new = F @ x_old`

### 4.3 观测模型（传感器）

我们的传感器只能测位置，测不了速度。所以观测向量只有 2 维：

```
z = [px_observed, py_observed]^T
```

状态到观测的映射用观测矩阵 H（代码第 110-113 行）：

```
H = [[1, 0, 0, 0],
 [0, 1, 0, 0]]
```

含义：`z = H @ x` -> "从 4 维状态中只取前两个（位置），忽略后两个（速度）"

### 4.4 噪声模型

**过程噪声 Q（代码第 116-117 行）：**
- 真实世界中，物体不会完美匀速——可能有风、地面不平、电机误差
- Q 矩阵量化了"运动模型和真实物理之间的偏差"
- Q 越大 -> 滤波器越不信任模型的预测

**测量噪声 R（代码第 119-120 行）：**
- 传感器的固有误差
- R 矩阵量化了"传感器的不可靠程度"
- R 越大 -> 滤波器越不信任传感器的读数

## 5. 卡尔曼滤波的五个公式（逐条拆解）

每一步滤波 = **预测**（公式 1-2）+ **更新**（公式 3-5）

### 阶段一：预测（我们猜物体现在在哪）

#### 公式 1：状态预测

```
x_k|_k_-_1 = F · x_k_-_1|_k_-_1
```

读法："用上一时刻的最佳估计 x_k_-_1|_k_-_1，通过运动模型 F，预测当前时刻的状态 x_k|_k_-_1"

| 符号 | 含义 |
|------|------|
| x_k|_k_-_1 | 在 k-1 时刻看 k 时刻的预测值（先验估计） |
| x_k_-_1|_k_-_1 | k-1 时刻的最佳估计（后验估计） |
| F | 状态转移矩阵，编码了运动模型 |

**代码：** `filters/kalman_filter.py:60` -> `self.x = self.F @ self.x`

#### 公式 2：协方差预测

```
P_k|_k_-_1 = F · P_k_-_1|_k_-_1 · F^T + Q
```

预测之后，不确定性**一定会增加**——因为模型不完美（Q），预测得越远越不确定。

**直觉**：你开车时闭眼 0.1 秒 -> 你基本知道车在哪。闭眼 5 秒 -> 你可能已经偏离车道了。不确定性随时间增长。

**代码：** `filters/kalman_filter.py:61` -> `self.P = self.F @ self.P @ self.F.T + self.Q`

---

### 阶段二：更新（传感器读数来了，修正预测）

#### 公式 3：卡尔曼增益

```
K = P_k|_k_-_1 · H^T · (H · P_k|_k_-_1 · H^T + R)^(-1)
```

这是卡尔曼滤波的灵魂。K 决定了"信模型"和"信传感器"之间的权重分配。

**极限情况分析：**

- 如果 **R -> 0**（传感器完美无误）-> `K ~= H^T · (H · H^T)^(-1)` -> 完全相信传感器
- 如果 **P -> 0**（模型极其确定）-> `K ~= 0` -> 忽略传感器，完全相信模型预测
- 正常情况：K 在两者之间自动调节

**代码：** `filters/kalman_filter.py:69-71` -> `K = self.P @ self.H.T @ np.linalg.inv(S)`

#### 公式 4：状态更新

```
x_k|_k = x_k|_k_-_1 + K · (z_k - H · x_k|_k_-_1)
 `--------T-------`
 创新（残差）
```

**残差 (innovation)** = 传感器实际读数 - 我们预测传感器应该读到的值。

- 如果残差大 -> 说明我们猜错了，需要大幅修正
- 如果残差小 -> 说明我们猜得准，小幅修正即可

K 决定了"用多大的力度去修正"。

**代码：** `filters/kalman_filter.py:73` -> `self.x = self.x + K @ y`

#### 公式 5：协方差更新

```
P_k|_k = (I - K · H) · P_k|_k_-_1
```

获得传感器信息后，不确定性**一定会降低**。降多少？看卡尔曼增益 K。

**代码：** `filters/kalman_filter.py:74` -> `self.P = (np.eye(4) - K @ self.H) @ self.P`

---

### 五公式总结（一页纸）

```
预测：
 (1) x = F·x "用模型猜下一时刻状态"
 (2) P = F·P·F^T + Q "预测的不确定性增加"

更新：
 (3) K = P·H^T·(H·P·H^T + R)^(-1) "算权重：信模型还是信传感器"
 (4) x = x + K·(z - H·x) "用传感器数据修正状态"
 (5) P = (I - K·H)·P "修正后不确定性降低"
```

### 6.2 初始化做了什么

```python
# filters/kalman_filter.py:30-43

self.F = [[1,0,dt,0], [0,1,0,dt], [0,0,1,0], [0,0,0,1]] # 匀速运动模型
self.H = [[1,0,0,0], [0,1,0,0]] # 只看位置，不看速度
self.Q = diag([0.01, 0.01, 0.05, 0.05]) # 过程噪声
self.R = diag([0.45^2, 0.45^2]) # 测量噪声

self.x = [0, 0, 0, 0]^T -> 初始猜测：在原点，速度为零
self.P = I_4 x 0.5 -> 初始不确定：半信半疑
```

初始 P 设得偏大（0.5）的好处：让滤波器在前几步快速改变估计值，而不是固执地停留在错误的初始猜测上。

### 6.3 一步滤波的完整流程

```python
# filters/kalman_filter.py:77-80

def step(self, z):
 self.predict() # (1) F -> x (2) F,P,Q -> P
 self.update(z) # (3) P,H,R -> K (4) K,残差 -> x (5) K,H -> P
 # 记录结果用于可视化
 self.estimates.append(x[:2]) # 只记录位置
 self.covariances.append(P.copy()) # 记录完整协方差
```

### 6.4 不确定性椭圆的原理

```python
# utils/visualization.py:23-33

def uncertainty_ellipse(cov_2x2, n_std=1.0):
 eigvals, eigvecs = np.linalg.eigh(cov_2x2) # 特征分解
 angle = arctan2(eigvecs[1,0], eigvecs[0,0]) # 椭圆方向
 width, height = 2 x n_std x sqrt(eigvals) # 椭圆尺寸
 return width, height, angle
```

协方差矩阵的**特征值** = 不确定性在各个方向上的"半径"
协方差矩阵的**特征向量** = 不确定性的"伸展方向"

1sigma 椭圆 = 物体有 68% 的概率落在这个椭圆内。

### 6.5 误差度量：RMSE

```
RMSE（均方根误差）= sqrt( 平均( (估计值 - 真实值)^2 ) )
```

`utils/visualization.py:print_metrics()` 计算并打印 RMSE。在 Demo 中：

- **观测 RMSE ~= 0.57m** — 你如果直接相信传感器，平均误差 57 厘米
- **滤波 RMSE ~= 0.25m** — 用了卡尔曼滤波，平均误差降到 25 厘米
- **误差降低 ~= 56%** — 滤波效果显著

## 8. 常见问题

### Q1: 卡尔曼滤波和"简单取平均"有什么区别？

简单取平均：对过去所有传感器读数取平均 -> 平滑但滞后严重。

卡尔曼滤波：不仅用历史信息，还用**运动模型**预测未来 -> 既平滑又无滞后。

### Q2: P 矩阵在什么时候会发散（越来越大）？

- 如果 **Q 太大而 R 太小** -> 滤波器过分相信传感器，P 不稳定
- 如果 **真实运动违背了模型假设** -> 比如物体突然以远大于 Q 预估的速度拐弯
- **数值不稳定** -> 长期运行中协方差可能因为浮点精度问题变成非正定，需要用 Joseph 形式或平方根滤波解决

### Q3: 为什么叫"滤波"而不叫"预测"？

历史上 "filter" 这个词来自信号处理。卡尔曼滤波确实可以同时做**滤波**（用当前及过去的观测估计当前状态）、**预测**（用过去的观测预测未来状态）和**平滑**（用全部观测估计过去某时刻的状态）。我们的 Demo 做的是"滤波"。

### Q4: 卡尔曼滤波和深度学习有什么区别？

| | 卡尔曼滤波 | 深度学习 |
|---|---|---|
| 需要大量数据 | 不需要 | 需要 |
| 需要运动模型 | 需要（手动设计） | 不需要（从数据中学习） |
| 可解释性 | 完全可解释 | 黑箱 |
| 计算量 | 极低 | 高 |
| 适合场景 | 实时状态估计、融合 | 模式识别、端到端控制 |

实际机器人系统中往往**两者结合使用**——比如用深度学习做目标检测，用卡尔曼滤波做检测结果的追踪和预测。

## 9. 当线性不够用 —— 扩展卡尔曼滤波（EKF）

### 9.1 线性 KF 有什么局限？

上一章我们假设传感器直接报出物体的 `[x, y]` 坐标。这就像一个 GPS 接收器。

但真实的传感器往往不会这么"贴心"。比如：

- **激光雷达（LiDAR）**：报的是"距离 + 方位角"——"3.5 米远，25 deg 方向"
- **毫米波雷达**：报的是"距离 + 相对速度"
- **RGB-D 相机**：报的是像素坐标 + 深度值

这些传感器的读数 **不是状态的线性函数**。距离 `r = sqrt(x^2 + y^2)` 里有平方和开根号，方位角 `theta = atan2(y, x)` 里有反正切——都是非线性运算。

**线性 KF 的 H 矩阵假设观测 = 常数 x 状态，在这里不成立。**

### 9.2 EKF 的核心思想：弯的当直的算

想象你站在一座小山坡上，想知道山的高度。山坡是弯的，但你只想在脚下这一小块区域内估计——你可以用脚下的**切线**来近似山坡：

```
       (真实曲线)
      /
     /  <-- 弯的
    /
---+---  <-- 在当前位置画切线（直的）
```

EKF 做的事一模一样：

1. 在当前状态估计点，对非线性函数 h(x) 做一阶泰勒展开
2. 泰勒展开 = "在这个点旁边，函数差不多就是一条直线"
3. 直线的斜率就是**雅可比矩阵**（导数推广到多维）

**直觉总结**：EKF = 标准 KF + 每步算一次雅可比矩阵 H_j 替代固定的 H。

### 9.3 雷达观测模型的雅可比

我们的 Demo 使用"距离 + 方位角"观测（模拟激光雷达）：

```
h(x) = [ range ] = [ sqrt(px^2 + py^2) ]
 [ bearing ] [ atan2(py, px) ]
```

这个函数在当前估计点 `(px, py)` 处的雅可比矩阵是：

```
H_j = [ drange/dpx drange/dpy 0 0 ]
 [ dbearing/dpx dbearing/dpy 0 0 ]

其中：
 r = sqrt(px^2 + py^2)
 drange/dpx = px / r <- "距离对 x 的变化率"
 drange/dpy = py / r <- "距离对 y 的变化率"
 dbearing/dpx = -py / r^2 <- "角度对 x 的变化率"
 dbearing/dpy = px / r^2 <- "角度对 y 的变化率"
```

`filters/extended_kf.py:52-67` 实现了这个计算。每步调用一次，每步的雅可比都不一样——因为"切线"取决于你在曲线上哪一点。

### 9.4 一个危险的坑：角度环绕

角度有一个反直觉的性质：**-179 deg 和 +179 deg 其实只差 2 deg**。

如果你预测物体在 179 deg 方向，传感器读到 -179 deg，简单相减得到 358 deg——滤波器会以为自己错了整整一圈，然后疯狂修正，直接发散。

修复方法：把角度差归一化到 `[-pi, pi]` 范围内：

```python
y[1, 0] = (y[1, 0] + np.pi) % (2 * np.pi) - np.pi
```

`filters/extended_kf.py:87` 这行代码保存了滤波器。

### 9.5 EKF 的优缺点

| 优点 | 缺点 |
|------|------|
| 可以处理非线性观测 | 需要手动推导雅可比（容易出错） |
| 计算量小（只多算一个雅可比） | 只做一阶近似，强非线性时误差大 |
| 工程中广泛使用（GPS、雷达跟踪） | 某些函数不可导时无法使用 |

## 10. 不要求导的聪明办法 —— 无迹卡尔曼滤波（UKF）

### 10.1 问题：雅可比不准怎么办？

EKF 的痛点是"用切线替代曲线"。当非线性很严重时（比如物体离传感器很近时，角度对位置极其敏感），一阶近似误差很大。

UKF 换了一套完全不同的思路：

> **与其把函数"掰直"，不如撒一批"侦察兵"去探探路。**

### 10.2 核心机制：Sigma 点

```
预测分布的均值 x_est, 协方差 P
 |
 v
 选 2n+1 个采样点（sigma 点）
 围绕均值，按协方差散开
 |
 v
 每个点分别通过非线性函数 h(x)
 （不需要近似！直接用真实函数）
 |
 v
 把回来后的点加权汇总
 -> 新的均值和协方差
```

**直觉类比**：

- EKF：派一个人去前方，他回来报告"前面的路大概是直的"（近似）
- UKF：派 9 个侦察兵分头走，回来汇总"路况全景"（采样）

这 9 个点（n=4 维状态，2n+1=9 个 sigma 点）是从均值和协方差中**确定性**选出来的——不是随机采样，而是精确按协方差的方向散开。

代码在 `filters/unscented_kf.py:82-100`。

### 10.3 无迹变换三步走

**第一步：选点。** 从 `(x, P)` 生成 sigma 点。

```
sigma[0] = x_mean <- 中心点
sigma[1..4] = x_mean + L 的各列 <- 沿协方差方向散开
sigma[5..8] = x_mean - L 的各列 <- 反方向对称
```

其中 L 是矩阵 `(n+lambda)P` 的平方根（用 Cholesky 分解求得）。

**第二步：传播。** 每个 sigma 点过一遍非线性观测函数 h(x)，得到 9 个"观测点"。

**第三步：汇总。** 加权平均算出预测观测的均值和协方差。

权重是预计算的，由三个参数决定：

| 参数 | 含义 | 推荐值 |
|------|------|--------|
| alpha | sigma 点散多开（大 -> 散得开） | 1.0 |
| beta | 利用高斯分布的先验知识 | 2.0 |
| kappa | 次级缩放（通常设 0） | 0.0 |

### 10.4 UKF 比 EKF 好在哪？

在 Demo 中运行两者，你会发现 UKF 的 RMSE 通常更低。原因：

- EKF：一阶精度（泰勒展开只保留一阶项）
- UKF：二阶精度（sigma 点能捕获分布的偏度/峰度）

**不需求导**是 UKF 的另一大优势——你不需要手算雅可比矩阵，只需要能"调用 h(x)"即可。

## 11. 终极武器 —— 粒子滤波（PF）

### 11.1 KF/EKF/UKF 的共有局限

前三个滤波器都默认一个前提：**状态的概率分布是高斯分布**（一个"钟形曲线"）。

```
 / \ <- 高斯：单峰，对称
 / \
 / \
```

但现实中很多情况不是这样。比如：

- 机器人在走廊里，它可能在左边房间，也可能在右边房间 -> **双峰分布**
- 刚启动时完全不知道自己在哪 -> **均匀分布**

粒子滤波完全放弃"必须是高斯"的假设。它用另一种方式表示概率分布：

> **用 N 个点（"粒子"）来近似分布。粒子稠密的地方 = 概率大。**

### 11.2 三步循环：预测 -> 加权 -> 重采样

Bootstrap 粒子滤波器（也叫 SIR：Sequential Importance Resampling）每步做三件事：

#### 第一步：预测（撒粒子）

每个粒子按运动模型 F 走一步，然后**加随机噪声**：

```
particle_i_new = F x particle_i_old + N(0, Q)
```

噪声至关重要——它保证粒子群保持"多样性"，不会被重采样耗光。

#### 第二步：加权（看谁更准）

传感器读数来了。对每个粒子问："如果物体真的在这个粒子的位置，传感器读到这个读数的概率有多大？"

```
w_i = exp(-0.5 x (真实观测 - 粒子预测的观测)^2 / 传感器噪声方差)
```

越接近真实观测的粒子，权重越大。**权重 = 似然 = "这个粒子有多靠谱"。**

代码在 `filters/particle_filter.py:73-93`，用对数域计算防止浮点下溢。

#### 第三步：重采样（优胜劣汰）

这是粒子滤波的灵魂操作。按权重重新分配粒子：

```
淘汰低权重粒子 -> 复制高权重粒子
```

想象一条线段，每段长度 = 粒子权重：
- 权重 0.4 的粒子占 40% 的线段 -> 大概率被抽中 200 次（在 500 个粒子中）
- 权重 0.001 的粒子占 0.1% -> 大概率被淘汰

代码用**系统重采样**（`systematic_resample`）：在 [0, 1/N] 随机选起点，每隔 1/N 取一个点，落在哪个粒子的段上就复制哪个。比纯随机更均匀。

### 11.3 粒子滤波的优缺点

| 优点 | 缺点 |
|------|------|
| 不假设高斯，能表示任意分布 | 计算量大（500 个粒子 x 120 步） |
| 不需求导，什么函数都能用 | 高维状态需要指数级粒子（维度灾难） |
| 能处理多模态（机器人定位经典场景） | 随机性导致每次运行结果不同 |
| 实现直觉简单 | 需要调粒子数和噪声参数 |

### 11.4 一个经典应用：Monte Carlo Localization (MCL)

粒子滤波最著名的应用是让机器人回答 **"我在哪？"**：

1. 开始时在整张地图上均匀撒 10000 个粒子（"我可能在任何一个位置"）
2. 机器人走一步（预测：所有粒子跟着走）
3. 激光雷达扫一圈（加权：和真实扫描匹配的粒子得高分）
4. 重采样（淘汰明显不对的区域）
5. 重复 2-4 -> 粒子逐渐聚集到真实位置

这就是 **AMCL（自适应蒙特卡洛定位）**，ROS 导航栈的标准组件。

## 12. 四大家族横向对比

### 12.1 一句话总结

| 滤波器 | 一句话 |
|--------|--------|
| **KF** | 线性系统 + 高斯噪声的最优解 |
| **EKF** | 把弯的当直的算，在当前位置画切线 |
| **UKF** | 派侦察兵（sigma 点）去探路，不画切线了 |
| **PF** | 撒一群蚂蚁（粒子），让它们自己找路 |

### 12.2 详细对比

| 特性 | KF | EKF | UKF | PF |
|------|:--:|:---:|:---:|:--:|
| 系统类型 | 线性 | 弱非线性 | 强非线性 | 任意 |
| 噪声假设 | 高斯 | 高斯 | 高斯 | 任意 |
| 计算量 | 极低 | 低 | 中 | 高 |
| 精度 | 最优（线性） | 一阶近似 | 二阶近似 | 渐近最优 |
| 需要求导 | 否 | **是** | 否 | 否 |
| 实现难度 | * | ** | *** | ** |
| 典型应用 | 导航融合 | GPS/雷达跟踪 | 姿态估计 | 机器人定位 |

### 12.3 如何选择？—— 决策指南

```
你的系统是线性的吗？
 |- 是 -> 用 KF（最简单，最准确）
 +- 否 -> 观测函数能求导吗？
 |- 能，且非线性不太强 -> 用 EKF
 |- 能，但非线性很强 -> 用 UKF
 +- 不能 -> 分布是高斯吗？
 |- 是 -> 用 UKF
 +- 否（多模态/任意分布）-> 用 PF
```

**实际工程建议**：起步选 UKF。不需求导，精度够好，计算量可接受。只有当你明确需要多模态分布（如全局定位）时才上 PF。
